## Running All Tests

```bash
# Run individual tests
for test in lattice-tests/test_*.py; do
    python "$test"
done
```

## Test Architecture

Each test follows this structure:

1. **Client Connection**: Connect to Moonshine lattice via routing tables
2. **Algorithm Loading**: Load algorithm specifications from `algorithm_examples/`
3. **Lattice Execution**: Run algorithm on selected lattice nodes using Aer simulator
4. **Verification**: Compare results to theoretical predictions
5. **Reporting**: Generate pass/fail status and detailed statistics

## Algorithm Specifications

Algorithm specifications are generated by `qbc_parser.py` and stored in `algorithm_examples/`:

- `grover_oracle_example.json` - Grover oracle for 3-qubit search
- `vqe_ansatz_example.json` - Hardware-efficient VQE ansatz
- `qft_example.json` - 4-qubit QFT circuit

## Adding New Tests

To add a new algorithm test:

1. Create `test_<algorithm>.py` in this directory
2. Implement test class inheriting structure from existing tests
3. Generate algorithm specification in `qbc_parser.py` algorithm library
4. Add test to this README

Example structure:

```python
class NewAlgorithmTest:
    def __init__(self, client: MoonshineClient):
        self.client = client

    def test_main_functionality(self):
        # Load algorithm spec
        # Run on lattice
        # Verify results
        pass

    def run_all_tests(self):
        # Execute test suite
        pass
```

## Performance Metrics

Each test tracks:
- **Execution time**: Wallclock time for algorithm execution
- **Shots**: Number of measurement samples
- **Success rate**: Percentage of trials meeting success criteria
- **Fidelity**: Quantum state fidelity compared to ideal
